# 06. 論点3：見積もりたたき（デザイン工程 × ページ数）v0.1

> 2026/6/11 たたき。Webの**デザインフェーズ**を「4工程 × ページ数」で積み上げる見積もり式。
> 単価は Notion「02_02 web制作サービス（ページデザイン費／ブランディング設計）」の既存値をアンカーに使用。
> ※ これは**デザインフェーズのみ**。実装は既存コーディング（px基準）、上流(事業/ブランド戦略)は別レイヤー。

---

## デザインの4工程（確定）

```
①情報整理 → ②設計 → ③ビジュアル要件 → ④デザイン
 構成設計    ワイヤー   トンマナ確定      UIカンプ
```
- **③ビジュアル要件 ＝ イラストのスタイルコードのWeb版**。「どう見せるか」を凍結する工程。
  ここが上流の尺度合わせ（`03_…` の Ds層を M3 に固める作業）。

## 工程の単位（ここが肝）

| 工程 | 単位 | 理由 |
| --- | --- | --- |
| ③ビジュアル要件（トンマナ） | **サイトに1回** | 一度決めれば全ページ共通 |
| ①情報整理・②設計・④デザイン | **ページごと** | ページ数 × そのページのLv で効く |

## 単価たたき表（税抜・Notion既存値ベース）

ページLvの目安：Lv1＝シンプル/参考あり、Lv2＝要素多め・軽い訴求、Lv3＝0ベース構成・訴求/演出重め。

## たたき式

```
デザイン総額 = ③ビジュアル要件(サイト1回) + Σ_各ページ ( ①+②+④ ) × そのページのLv
            （＋必要ならコピー、＋特急1.3 / 調整費2,000円/日）
```

### 計算例

**例1：LP 1枚（トンマナLv1・ページLv2）**
```
③15,000 + ページ55,000 = ¥70,000
```

**例2：コーポレート5ページ（トンマナLv2／TOPだけLv3・下層4pはLv1）**
```
③40,000 + TOP 85,000 + 下層 35,000×4(=140,000) = ¥265,000
```

**例3：サービスサイト3ページ（トンマナLv2／全ページLv2）**
```
③40,000 + 55,000×3(=165,000) = ¥205,000
```

→ ページ数とLvを入れるだけで**機械的にたたける**。コーディングの「ページLv×レスポンシブ」と
同じ発想を、デザインフェーズにも入れた形。
