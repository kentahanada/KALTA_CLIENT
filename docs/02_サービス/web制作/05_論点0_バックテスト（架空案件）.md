# 05. 論点0 バックテスト（架空案件）v0.1

> 2026/6/11 思考実験。二軸モデル（サクセスLv × 成熟度コード）と判定ロジックを架空案件に通し、
> **破綻・穴を炙り出す**のが目的。参照：`03_…(尺度コード)`／`04_…(変換ロジック)`。

各ケースの流れ：**初期シグナル→一次サクセスLv判定→（有償診断）→尺度コード→マトリクス判定→
エントリーモジュール→ロードマップ→所見**。

---

## ケース1：ふわふわ高サクセス（CONVERT主戦場）

- 客：地方の個人経営カフェ（2店舗）オーナー。「インスタはやってる、常連はいるが新規が来ない。
  ちゃんとしたサイトで"うちらしさ"を出したい。予算は出す、まず相談したい」。
- 一次判定：成果(新規集客)志向＋「予算は出す」＋相談姿勢 → **経営者タイプ・サクセスLv4見込み**。
- 有償診断 → 尺度コード：
  `G1,2 / F2 / Sc2 // Bz:M2 Br:M1 IA:M0 Ds:M0`
  （事業・客層は分かっている＝Bz:M2／"うちらしさ"が未言語化＝Br:M1／構成・デザインは未着手）
- マトリクス：**Lv4 × 成熟度低 → CONVERT**。
- エントリー判定：上流から最初の M<2 ＝ **Br（Bzはクリア）→ ブランド設計（思考難度Lv4）**。
- ロードマップ：診断 → ブランド設計(Br M1→M3) → 構成設計(IA) → デザイン(Ds) → 実装(Cd)〔→ SEO運用〕。
  診断費は本契約に充当。
- 所見：◎ ふわふわを門前払いせず、ブランド設計という上流から正当に着金。設計どおり流れる。

## ケース2：カンプ支給（EXECUTE＝既存コーディング）

- 客：Web制作会社の外注ディレクター。「Figmaカンプ完成・10P・WP化・レスポンシブ有・納期2週」。
- 一次判定：ドキュメント主義・指示明確 → **エージェンシータイプ・Lv4**。
- 尺度コード：`G3 / F2 / Sc3 // Bz:- Br:- IA:M3 Ds:M3`（カンプ確定＝ゲート通過）。
- マトリクス：**Lv4 × 成熟度高 → EXECUTE**。エントリー＝**実装(Cd)直行**。
- 所見：◎ 既存コーディングサービスと完全一致。フレームが現行の成功パターンを包含することを確認。

## ケース3：予算先行・相見積もり（罠 → 踏み絵）

- 客：「個人事業主、5万くらいでいい感じのHPを、参考はこれ、できるだけ早く。他社さんも見てます」。
- 一次判定シグナル：「いい感じ」「5万で」「早く」「他社も」→ **予算先行＋感覚派＋相見積もり＝Lv2（摩耗型）**。
- 対応：いきなり断らず**有償診断を提示（踏み絵）**。
  - 反応A：診断にも金を出さない／ゴネる → **DECLINE（自然脱落）**。
  - 反応B：診断を買う → 実はLv3だった可能性（踏み絵突破）→ 受けて成熟度判定へ。
- 所見：△→修正。**Lv2を機械的にDECLINEにするのは粗い**。有償診断が踏み絵として働くので、
  「Lv2は診断で踏ませる／拒否ならDECLINE」が正しい（→ 修正B）。

## ケース4：高サクセスだが「実行だけ」希望（穴を発見）

- 客：技術に強い事業主。「自分でワイヤーまで作った。デザインはテンプレ購入済みで素材ある。
  コーディングだけ頼みたい。ブランドは確立済み」。→ **技術オタクタイプ・Lv5**。
- 尺度コード：`G3 / F2 / Sc2 // Bz:M3 Br:M3 IA:M2 Ds:M2`（Dsはテンプレ素材だが"実装可能な確定カンプ"ではない）。
- 実装(Cd)の直上＝Ds。ゲート「Ds=M3必須」に**未達** → CONVERT(デザイン仕上げ/カンプ化)を提案。
- だが客「デザインに金は払いたくない、素材あるんだから実装して」。
- ここで**穴**：CONVERTを断られたらDECLINE？ 客はLv5の優良客なのに？
- 解：未成熟層の解消は「**自社が売る(CONVERT)**」だけでなく「**客が凍結資産を自前/外部で用意**」でもよい。
  Dsを実装可能なM3まで客に持ってきてもらえば **EXECUTE可**。持ってこれない/曖昧なまま強行希望なら
  リスク明示＋着手金。→ **コーディングの"カンプ支給"はまさにこの後者ルート**（→ 修正A）。
- 所見：穴発見。ゲートの条件は「**層がM3になること**」であって「自社がCONVERTを売ること」ではない。

## バックテストで見つけた穴と修正

### 修正A：ゲート達成は2ルート（自社CONVERT / クライアント調達）
- ゲートの条件＝「直上の層がM3（デザイン）/M2（その他）に達すること」**だけ**。
- 達成手段は2つ：**①自社の構造化モジュールを売る(CONVERT)** ／ **②クライアントが自前・外部で
  凍結資産を用意する**。コーディングのカンプ支給は②。CONVERTは"提案"であり強制ではない。
- → `04_…` §5 に追記。

### 修正B：サクセスLv1-2は一律DECLINEにせず精緻化
- **Lv2 → 有償診断（踏み絵）に乗せる**。買えば受ける（防御＝着手金・線引き・有料ディレクション）。
  拒否・ゴネで DECLINE。
- **Lv1 → DECLINE**（搾取型）。
- → `04_…` §1.5 マトリクスを精緻化。

### 修正C：サクセスレベルは2段判定
- **一次判定（無償・初期シグナル）**：初期メール/問い合わせの言い回しで粗くLvを当てる
  （Notion「05_…」の見抜きポイント表を使用）。
- **確定判定（有償診断内）**：診断の中で成熟度コード化とともにLvを確定。
- Lv1-2濃厚なら**診断前に線引き**（撒き餌診断はしない）。→ `04_…` §8 残論点③を解消。

## 結論

二軸モデルは**3/4ケースでそのまま機能**し、現行コーディングの成功（ケース2）も包含。
ケース3・4で出た穴は**修正A/B/Cで吸収可能**＝フレームの土台は妥当。次は修正を反映し、論点3（価格）へ。
