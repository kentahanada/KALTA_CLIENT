# 03. イラスト制作スタイル認識合わせ SOP

> 出典：Notion「イラスト制作スタイル認識合わせ SOP（7軸版・正式）」
> フレーム：**DSP（定義 → 手順 → 判定）** ／ ステータス：公開

スタイルは「線・形・塗りの要素の掛け算」で決まる。感覚的指示を禁止し、
**スタイルをコード化**して認識ズレと修正工数を最小化する。

---

## 0. 目的・前提

- 「◯◯風」「雰囲気こんな感じ」といった**感覚的指示を禁止**する。
- 制作前にスタイルを**コード化・固定**し、認識ズレと修正工数を最小化する。
- 社内・外注を問わず、**誰が描いても同じスタイル判断に到達できる状態**を作る。
- 適用範囲：キャラクターイラスト全般（立ち絵／SD／グッズ／SNS／挿絵）、社内制作・外注発注・教育/品質管理。

---

## 1. 定義（Definition）：スタイルは軸で定義する

イラストの見た目はセンスではなく、以下の要素の組み合わせで再現可能に定義できる。
（タイトルは「7軸版」だが、影・陰影まわりを分解し実質9要素で運用）

```
スタイル ＝
 ① 線のラフ度(L) × ② 主線の太さ(T) × ③ 線の強弱(W) × ④ デフォルメ度(D)
 × ⑤ 塗り技法(P) × ⑥ 色彩設計(C) × ⑦ 影・光設計(S)
 × ⑧ 陰影モデル(H：Form Shading) × ⑨ 影境界(K：Shadow Edge)
```

### 線・形レイヤー
| 軸 | ラベルと定義 |
| --- | --- |
| ① 線のラフ度 (L) | L0=線なし／L1=ラフ線（揺れ・描き直しあり）／L2=クリーン線（ペン入れ済・揺れなし） |
| ② 主線の太さ (T) | T1=細線（繊細・大人向け）／T2=標準／T3=太線（ポップ・視認性重視）※主線なしは不可 |
| ③ 線の強弱 (W) | W0=均一線（セル画・アニメ系）／W1=強弱あり（イラスト・質感重視） |
| ④ デフォルメ度 (D) | D0=リアル寄り（高頭身）／D1=標準／D2=デフォルメ（比率は維持）／D3=SD（2〜3頭身） |

### 塗りレイヤー
| 軸 | ラベルと定義 |
| --- | --- |
| ⑤ 塗り技法 (P) | P1=セル塗り／P2=厚塗り／P3=フラット塗り／P4=水彩・テクスチャ |
| ⑥ 色彩設計 (C) | C1=高彩度・ポップ／C2=標準／C3=低彩度・くすみ／C4=モノトーン/制限配色 |
| ⑦ 影・光設計 (S) | S0=影ほぼ無し／S1=影1段（セル影）／S2=多段影・グラデ／S3=演出的ライティング（逆光・縁光） |
| ⑧ 陰影モデル (H) | H0=指定なし／H1=最小／H2=陰影あり（面の向きで立体）／H3=陰影強め |
| ⑨ 影境界 (K) | K0=指定なし／K1=面影（硬い境界）／K2=丸影（柔らかい境界）／K3=混在OK |

> ⑦S は「影・光の処理レベル（段数/グラデ/演出光）」、⑧H は「面の向きによる立体感（フォームシャドウ）」、
> ⑨K は「落ち影の境界形状（エッジ）」。混同しないこと。

---

## 2. 手順（Process）

### スタイルコードの作成
1. 各軸すべてにラベルを割り当てる。
2. 以下形式でコード化する。

```
L?-T?-W?-D? / P?-C?-S?-H?-K?
```

### コード例
| コード | 意味 |
| --- | --- |
| `L2-T2-W0-D1 / P1-C2-S1-H2-K1` | TVアニメ風 / ソシャゲ立ち絵 |
| `L1-T1-W1-D2 / P4-C3-S0` | 絵本・インディーズ・柔らかい印象 |
| `L2-T3-W0-D3 / P3-C1-S0` | ポップSD / グッズ・アイコン向け |

### 実務フロー
- **依頼側（上流）**：全軸を指定する。不明な場合は「既存スタイルコード」を指定しても可。
- **制作側（下流）**：コードを確認 → 近似サンプルを提示 → 認識一致を確認後に制作開始。

---

## 3. 判定（Decision）

| 判定 | 基準 | 次の行動 |
| --- | --- | --- |
| OK | 全軸一致 | 本制作へ進行 |
| NG | 1軸でもズレ | ラフ段階で修正 |

### よくあるミス
- 「◯◯風」だけ指定してコードが無い
- P（塗り）だけ指定し、C・Sを決めていない
- ラフでスタイル確認せず清書に入る（※清書後のスタイル修正は原則NG）

---

## 4. 完了条件

- スタイルコードが明示されている
- 上流・下流で同一コードを確認済み
- 参考画像が共有されている

→ この3点を満たした時点で「スタイル認識合わせ完了」。
