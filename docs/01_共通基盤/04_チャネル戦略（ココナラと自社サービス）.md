# 04. チャネル戦略（ココナラ と 自社サービスの差異）

> 2026/6/11 言語化。**サービス設計思想とチャネルは別レイヤー**である、という整理。

---

## 大原則：設計思想はチャネル非依存

サービスの設計思想——**レベル(Lv)・クロスセル・工程モジュール・共通業務フロー(S0〜S6)・
構造化見積もり（カテゴリ×Lv×工数係数）**——は、**どのチャネルで売っても同じ**。
（→ 思想の核は `00_設計思想（共通の根っこ）.md`）

**チャネルによって変わるのは「中身」ではなく「経済性（客単価・利益率）と役割」だけ。**

> 🪄 同じサービスでも、チャネルを自社に移すと高値で売れる——という"魔法"。
> これは中身が変わったからではなく、**チャネルが価格の天井と信頼の出どころを決めている**ため。

---

## ココナラ（スキルマーケット）

| 観点 | 内容 |
| --- | --- |
| 強み | 集客・決済・信頼担保（評価/実績）を**プラットフォームが肩代わり**してくれる |
| 弱み | **薄利・低客単価**（手数料〜22%）。匿名性ゆえ「知らない人への不安」が価格を抑える |
| 客層 | 個人・低〜中単価層が中心（例：個人Vtuber） |
| 役割 | ① **サービスのスモールテスト**（新メニューの反応を低リスクで検証）<br>② **低単価層へのアプローチ**<br>③ **実績・評価の蓄積**（信頼資産づくり） |

- 実データ：主力はキャラクターイラスト ¥30,000（→ `02_サービス/イラスト制作_ヨンマルヨン/07_…`）。
- 限界の実証：個人Vtuber × 高単価（¥126,000）は失注（→ `08_案件ログ（受注・失注）.md`）。
  **客単価の天井が低い**ことの裏付け。

---

## 自社サービス（株式会社カルタとして請ける）

| 観点 | 内容 |
| --- | --- |
| 強み | 法人の信頼・ブランド・**直接契約**。手数料ゼロ。**高客単価でやれる** |
| コスト | 集客・信頼担保・決済・受発注の仕組みを**自前で用意**する必要がある |
| 客層 | 法人・継続・大型・toB、客単価を上げたい案件 |
| 役割 | ① **客単価を上げたい時**<br>② **法人（株式会社カルタ）として請けたい時**<br>③ 継続・大型・toB案件 |

- Web側は自社ブランド「ピクセルスタジオ」で先行（→ `02_サービス/web制作/00_サービス概要.md`）。
- イラスト側はまだ自社サービスサイトを持たずココナラ中心（Webの知見を横展開できる関係）。

---

## 使い分けの指針

| 目的・状況 | 推奨チャネル |
| --- | --- |
| 新メニューの反応を低リスクで試す | ココナラ（スモールテスト） |
| 低単価層・個人をまず取りたい | ココナラ |
| 実績・評価を貯めたい | ココナラ |
| 客単価を上げたい | 自社サービス |
| 法人として・継続/大型/toBで請けたい | 自社サービス |

> ⚠️ ブランドの線引き：ヨンマルヨンは現状**アンダーグラウンド扱い**（カルタ表ポートフォリオにしない）。
> イラストを自社・高単価チャネルへ広げる際は、ブランドの出し方が論点になる（→ `03_将来構想/`）。

---

## まとめ（1行）

**「設計思想（中身）はどこでも一緒。ココナラ＝薄利だが手間いらずでテスト/低単価向け、
自社＝手間はかかるが高単価/法人向け」** —— チャネルは価格と信頼の器であって、サービス設計とは別レイヤー。
