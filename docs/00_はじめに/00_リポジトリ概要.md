# 00. リポジトリ概要

## このリポジトリの目的

株式会社カルタのクライアントワーク系サービスについて、散在している情報
（Google ドキュメントの対応マニュアル、Notion のココナラ関連ドキュメント、MTGログなど）を
**一箇所に体系化し、属人化を防ぐ**ことを目的とします。

当面の中心は、デジタルクリエイション部門がココナラで運用しているイラスト制作サービス
**「ヨンマルヨン（404）」** です。

## 設計の基本思想（最重要）

カルタのクライアントワークは、**共通の根っこ（設計思想・業務フロー・見積もり設計）** を持ち、
そこからサービスごとに詳細が枝分かれする、という構造で設計されています。

```
            ┌─────────────────────────────┐
            │  01_共通基盤（根っこ）          │
            │  ・設計思想                    │
            │  ・共通業務フロー（S0〜S6）      │
            │  ・見積もり設計フレーム          │
            │  ・調整費／トラブル対応          │
            └──────────────┬──────────────┘
                           │ 詳細が枝分かれ
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
  イラスト制作         Web制作          （将来）デジタルコンテンツ
  （ヨンマルヨン）                       制作サービスへの昇華
```

- **Web制作とイラスト（デジタルコンテンツ）制作は兄弟**。根っこの思想は共通。
- 見積もりはどちらも「**カテゴリ × レベル(Lv) × 工数係数 × 価格目安**」という共通フレームで組む。
- 業務フロー（問い合わせ→ヒアリング→見積もり→契約→制作→確認→納品）も共通の骨格を持つ。

## 読む順番（おすすめ）

1. `docs/01_共通基盤/00_設計思想（共通の根っこ）.md` … まず思想を理解する
2. `docs/01_共通基盤/01_共通業務フロー.md` … 全サービス共通の進め方
3. `docs/01_共通基盤/02_見積もり設計フレーム.md` … 見積もりの作り方の考え方
4. `docs/02_サービス/イラスト制作_ヨンマルヨン/00_サービス概要.md` … ヨンマルヨンとは何か
5. 以降、各サービスの料金・フロー・SOP・記録へ

## メンテナンス方針

- 一次情報（Notion / Google Drive）が正。本リポジトリはその**整理スナップショット**。
- 大きな方針変更があったMTGは `02_サービス/.../records/` に日付付きで残す。
- 用語・命名で迷う点は `01_命名・フォルダ構成の方針（要検討メモ）.md` に集約する。
